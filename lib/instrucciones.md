persistència i serveis externs Temporització4 hores TecnologiesFlutter · Firebase Authentication · Cloud Firestore · Android Studio ModalitatIndividual 📝 ENUNCIATL'objectiu d'aquesta pràctica és construir una aplicació Flutter completa que combini l'Autenticació amb Firebase i la Persistència de dades amb Cloud Firestore.L'aplicació és una Llista de Tasques (To-Do List) on cada usuari registrat tindrà la seva pròpia llista privada.Quan un usuari inicia sessió, veurà únicament les seves tasques.Si un altre usuari entra des del mateix dispositiu, veurà les seves pròpies dades.En tancar sessió, l'aplicació tornarà a la pantalla de login.🎯 Idea clau: L'aplicació és la unió de la pantalla de login del recurs d'autenticació i la llista de tasques del recurs de Firestore, assegurant que cada usuari tingui les seves dades separades.🎯 OBJECTIUS D'APRENENTATGEIntegrar Firebase Authentication i Cloud Firestore en una mateixa aplicació.Crear col·leccions Firestore per usuari, separant les dades de cada compte.Aplicar el patró StreamBuilder per gestionar l'estat d'autenticació.Implementar el flux complet de login, registre i logout.Aplicar operacions CRUD (Create, Read, Update, Delete) sobre Firestore.✅ REQUISITS FUNCIONALSAutenticacióPantalla de login amb camps d'email i contrasenya.Botons per iniciar sessió i per canviar a mode registre.Validació de camps: no buits i format d'email vàlid.Missatges d'error clars per a credencials incorrectes.Botó de logout visible a la pantalla principal.Llista de TasquesCol·lecció de tasques pròpia per a cada usuari.Visualització en temps real de les tasques de l'usuari.Afegir tasques noves.Marcar tasques com a completades (canvi visual com text ratllat).Esborrar tasques.Ordenació per data de creació (les més recents primer).🏗️ ARQUITECTURA DE L'APLICACIÓFitxerResponsabilitatmain.dartPunt d'entrada. Inicialitza Firebase i gestiona la navegació reactiva amb StreamBuilder.screens/login_screen.dartFormulari de login i registre. Crida als mètodes d'AuthService.screens/home_screen.dartPantalla principal amb la llista de tasques i el botó de logout.services/auth_service.dartEncapsula tota la lògica de Firebase Authentication.services/firestore_service.dartEncapsula les operacions CRUD sobre la col·lecció de tasques a Firestore.🚀 DESENVOLUPAMENT PAS A PASPAS 1 — Crear el projecte i l'estructuraExecuta al terminal:Bashflutter create todo_fire_auth
Crea la següent estructura dins de lib/:main.dartscreens/login_screen.dartscreens/home_screen.dartservices/auth_service.dartservices/firestore_service.dartPAS 2 — Configurar FirebaseA la Firebase Console:Authentication: Activa el mètode Correu electrònic/contrasenya.Cloud Firestore: Crea la base de dades en Mode de prova i tria la ubicació europe-west.PAS 3 — Configurar FlutterFire CLIDins la carpeta del projecte, executa:Bashflutterfire configure
PAS 4 — Afegir dependènciesAfegeix al pubspec.yaml:YAMLdependencies:
  firebase_core: ^3.0.0
  firebase_auth: ^5.0.0
  cloud_firestore: ^5.0.0
Després executa flutter pub get.PAS 5 — Crear auth_service.dartAquest servei gestiona el registre, login i logout utilitzant firebase_auth.PAS 6 — Crear firestore_service.dart ★ Peça clauUsem l'UID de l'usuari per separar les dades.
Estructura: usuaris/{uid}/tasques/{idTasca}.PAS 7 & 8 — Interfície d'usuari (UI)LoginScreen: Reutilitza el codi de validació i camps d'email/contrasenya.HomeScreen: Mostra la llista usant un StreamBuilder que connecta amb firestore_service.dart.
